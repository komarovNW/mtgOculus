# Документация проекта

Этот файл нужен как карта документации. После серии изменений по UI, авторизации, деплою и backend-контрактам в репозитории остались как актуальные документы, так и старые MVP-черновики. Ниже зафиксировано, чем пользоваться сейчас.

## Актуальные документы

Если нужно понять проект в его текущем состоянии, начинайте отсюда:

- `README.md`
  - общее описание проекта;
  - запуск;
  - env;
  - основные маршруты;
  - временная mock-авторизация;
  - ссылки на остальные docs.
- `readme/BACKEND_API_HANDOFF.md`
  - главный контракт между фронтом и будущим backend;
  - текущие ожидания по данным, ручкам, auth, ошибкам, сортировкам и импорту турнира.
- `readme/ROUTES.md`
  - все текущие маршруты;
  - какие страницы публичные;
  - где нужен вход;
  - как устроена навигация.
- `readme/PROJECT_STRUCTURE.md`
  - актуальная структура `src/`;
  - зоны ответственности директорий и файлов.
- `readme/UI_GUIDELINES.md`
  - текущая UI-логика;
  - светлая и тёмная темы;
  - token-система;
  - требования к таблицам, карточкам, графикам и текстам.
- `readme/ENV_EXAMPLE.md`
  - все env-переменные;
  - mock/backend сценарии;
  - временные auth-переменные.
- `readme/API_INTEGRATION_CHECKLIST.md`
  - что проверить при переходе с моков на реальный backend.
- `readme/TIMEWEB_DEPLOY.md`
  - как настроен и обновляется деплой на Timeweb;
  - когда срабатывает автодеплой после push.

## Что считать источником правды

Если документы противоречат друг другу, используйте такой приоритет:

1. текущий код в `src/`;
2. `readme/BACKEND_API_HANDOFF.md`;
3. `README.md`, `readme/ROUTES.md`, `readme/PROJECT_STRUCTURE.md`, `readme/UI_GUIDELINES.md`;
4. остальные md-файлы.

## Для разработчика backend

Если задача связана с backend или refactor интеграции, практический минимум такой:

1. Прочитать `README.md`.
2. Прочитать `readme/BACKEND_API_HANDOFF.md`.
3. Прочитать `readme/API_INTEGRATION_CHECKLIST.md`.
4. При необходимости свериться с `src/shared/api/types.ts` и страницами в `src/pages/`.

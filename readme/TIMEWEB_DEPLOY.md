# Деплой в Timeweb App Platform

Актуально для проекта `magic_oculus`, репозиторий:

```text
https://github.com/komarovNW/mtgOculus.git
```

## Рекомендуемый сценарий

Для первого деплоя лучше использовать `App Platform -> Frontend -> React`.

Есть два способа подключить репозиторий:

- через подключение аккаунта `GitHub` в панели Timeweb;
- через `git-репозиторий по URL`.

Важно:

- если подключать репозиторий по `URL`, автодеплой после новых коммитов недоступен;
- если подключить именно аккаунт `GitHub`, Timeweb сможет отслеживать новые коммиты и делать автодеплой.

## Параметры приложения

При создании приложения указывай:

- тип приложения: `Frontend`
- фреймворк: `React`
- ветка: `main`
- Node.js: `22`
- build command: `npm run build`
- output directory: `dist`

## Переменные окружения

### Временный запуск на моках

Для первого публичного запуска без backend:

```env
VITE_API_BASE_URL=http://localhost:8000/api/v1
VITE_USE_MOCKS=true
VITE_MOCK_SCENARIO=default
```

### Когда появится backend

```env
VITE_API_BASE_URL=https://your-api-domain/api/v1
VITE_USE_MOCKS=false
```

## Пошагово в панели Timeweb

1. Открой `App Platform`.
2. Нажми `Создать`.
3. В блоке репозитория выбери один из вариантов:
   - `Добавить аккаунт -> GitHub`, если нужен автодеплой;
   - `Добавить аккаунт -> Подключите git-репозиторий по URL`, если нужен быстрый ручной запуск.
4. Если идешь через URL, вставь:

```text
https://github.com/komarovNW/mtgOculus.git
```

5. Выбери ветку `main` и последний коммит.
6. Укажи параметры сборки из раздела выше.
7. В блоке `Переменные` добавь env-переменные для mock-режима.
8. Запусти деплой.

## Что будет после деплоя

- Timeweb автоматически выдаст бесплатный технический домен;
- для него автоматически выпустится `Let's Encrypt` SSL-сертификат;
- по этому домену можно сразу проверить сайт.

## Подключение своего домена

После успешного деплоя:

1. Перейди в `Домены и SSL`.
2. Открой нужный домен.
3. На вкладке `DNS` добавь `A`-запись.
4. В поле `Привязать к сервису` выбери наше frontend-приложение.

Важно:

- для frontend-приложения домен должен быть добавлен в панель Timeweb;
- домен должен использовать `NS`-серверы Timeweb.

## Что делать после первого деплоя

Если приложение создано через прямое подключение `GitHub`, оставь включенной опцию автоматической сборки по последнему коммиту.

Если приложение создано по `URL`, новые коммиты придется деплоить вручную через вкладку `Деплой`.

# UI Guidelines for MTG Global Stats MVP

## Общий стиль

Сайт должен выглядеть как чистый аналитический dashboard для статистики Magic: The Gathering.

Ключевые качества UI:

- читаемость;
- простота;
- аккуратные таблицы;
- понятные фильтры;
- быстрый доступ к турнирам, игрокам и колодам;
- минимум декоративного шума.

## Язык интерфейса

Весь интерфейс на русском языке.

Примеры:

- `Турниры`
- `Игроки`
- `Колоды`
- `Метагейм`
- `Матчапы`
- `Малая выборка`
- `Пока нет данных`

## Layout

Базовая структура страницы:

```text
Header / Navigation
Main content container
Page title
Filters, если нужны
Summary cards
Main sections / tables
```

Рекомендуемая максимальная ширина контента: `1200px` или близко к этому.

Контент должен быть центрирован.

## Цвета

Для MVP не нужно сложное брендирование.

Рекомендуется:

- светлый фон;
- белые карточки;
- нейтральные серые border/background;
- один основной accent color для ссылок, кнопок и активных элементов;
- не использовать слишком много ярких цветов.

Если выбирается Tailwind, использовать стандартную палитру и не создавать сложную кастомную тему без необходимости.

## Типографика

Требования:

- крупные заголовки страниц;
- понятные подписи карточек;
- числовые значения в summary cards должны быть заметными;
- таблицы должны быть легко читаемыми.

Рекомендуемые размеры:

- Page title: 28–32px;
- Section title: 20–24px;
- Table text: 14–16px;
- Helper text: 12–14px.

## Общие UI-компоненты

Агент должен создать и использовать базовые компоненты:

```text
Button
Card
Table
Badge
StatCard
Select
Input
FileInput
Tabs
PageHeader
EmptyState
ErrorState
LoadingState
```

## Summary cards

Используются на главной, странице игрока, странице колоды и странице турнира.

Пример карточек:

```text
Турниров: 5
Игроко-участий: 128
Матчей: 240
Уникальных колод: 26
```

Требования:

- одинаковый размер в рамках одной строки;
- короткий label;
- крупное значение;
- поддержка пустого значения `—`.

## Таблицы

Таблицы — основной способ отображения данных.

Общие требования:

- заголовки колонок фиксированно понятные;
- строки должны иметь hover state, если они кликабельны;
- player, deck и tournament должны быть ссылками, если есть id;
- пустые значения показывать как `—`;
- проценты показывать с одним знаком после запятой.

Пример форматирования процентов:

```text
61.1%
```

Не показывать слишком много знаков:

```text
61.111111111%
```

## Мобильная версия

На мобильных устройствах таблицы не должны ломать layout.

Допустимые варианты:

1. horizontal scroll для таблицы;
2. карточное представление строк;
3. скрытие второстепенных колонок.

Для MVP достаточно horizontal scroll.

## Badges

Использовать badges для:

- формата: `Legacy`;
- типа турнира: `Дейлик`, `Турнир`;
- малой выборки: `Малая выборка`;
- record: `4-0`, `3-1`.

## Loading state

Каждая страница должна иметь loading state.

Не оставлять пустой белый экран.

Допустимо:

- simple spinner;
- skeleton cards/tables;
- текст `Загрузка...`.

## Empty state

Если данных нет, показывать понятный текст.

Примеры:

```text
Пока нет загруженных турниров по Legacy в Москве.
```

```text
По выбранным фильтрам нет данных.
```

## Error state

Если API вернул ошибку, показывать:

- короткое понятное сообщение;
- кнопку `Повторить`, если это имеет смысл.

Пример:

```text
Не удалось загрузить статистику. Попробуйте обновить страницу.
```

## Фильтры

Фильтры должны соответствовать сущностям при создании турнира:

- город;
- клуб;
- формат;
- тип турнира;
- период.

Для MVP основной дефолт:

```text
Город: Москва
Формат: Legacy
Клуб: все
```

Фильтры должны проектироваться так, чтобы их можно было включить позже без переписывания страниц.

## Диаграммы

Если используются диаграммы:

- для метагейма лучше bar chart, а не pie chart;
- не использовать сложные цвета;
- диаграмма должна дублироваться таблицей или сопровождаться таблицей;
- не показывать график, если данных недостаточно.

## Кликабельность

Все сущности с id должны быть кликабельными:

```text
Турнир -> /tournaments/:id
Игрок -> /players/:id
Колода -> /decks/:id
```

Если id нет, элемент не должен выглядеть как ссылка.

## Что не делать

Не делать перегруженный UI.

Не использовать много цветов для winrate.

Не делать сложные анимации.

Не использовать непонятные сокращения без необходимости.

Не скрывать важные данные только в tooltip.

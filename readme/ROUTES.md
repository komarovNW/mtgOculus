# Routes Documentation

Этот документ описывает текущие маршруты проекта `Magic Oculus` и правила доступа к ним.

## Публичные маршруты

Эти страницы доступны без авторизации.

### `/`

Главная страница статистики.

Показывает:

- summary cards;
- последние турниры;
- метагейм по колодам;
- результативность колод;
- игроков с лучшими результатами;
- частые матчапы.

По умолчанию фронт открывает статистику для:

```text
cityId = moscow
formatId = legacy
clubId = not passed
```

### `/home`

Технический legacy-route. Сейчас делает редирект на `/`.

### `/tournaments`

Список турниров с фильтрами по:

- городу;
- клубу;
- формату;
- типу турнира;
- периоду.

### `/tournaments/:id`

Детальная страница турнира.

Показывает:

- общую информацию о турнире;
- итоговые стендинги;
- раунды и паринги;
- список колод участников;
- метагейм турнира.

### `/players`

Список игроков с фильтрами и сортировками.

Поддерживаются:

- поиск по имени;
- сортировка по результатам;
- сортировка по количеству матчей и турниров;
- фильтры по тем же справочникам, что и на других публичных экранах.

### `/players/:id`

Детальная страница игрока.

Показывает:

- общую статистику;
- турниры игрока;
- колоды игрока;
- последние матчи.

### `/decks`

Список колод с фильтрами и сортировкой.

### `/decks/:id`

Детальная страница колоды.

Показывает:

- общую статистику колоды;
- результаты по турнирам;
- игроков, которые играли этой колодой;
- матчапы.

### `/login`

Страница входа в служебный раздел.

Сейчас используется только для доступа к экрану добавления турнира. Публичная статистика входа не требует.

## Защищённый маршрут

### `/admin/tournaments/create`

Экран ручного импорта турнира.

Сейчас для него нужна авторизация с permission:

```text
tournament:create
```

Если пользователь не авторизован, фронт переводит его на `/login`.

Форма сейчас содержит:

- `date`
- `cityId`
- `clubId`
- `tournamentType`
- `formatId`
- `aetherhubUrl`
- `finalStandingsFile`
- `allRoundsFile`
- `playerDecksText`

## Навигация в шапке

Всегда видны:

- `Главная`
- `Турниры`
- `Игроки`
- `Колоды`

Условия:

- ссылка `Добавить турнир` показывается только авторизованному пользователю с permission `tournament:create`;
- кнопка `Войти` показывается гостю;
- кнопка `Выйти` и имя пользователя показываются после входа;
- в шапке также есть переключатель `Светлая / Тёмная`.

## 404

Все неизвестные маршруты попадают на `NotFoundPage`.

## Что важно для backend

- все `id` из API используются прямо в URL;
- публичные страницы не требуют auth;
- защищён только импорт турнира;
- наличие `Authorization` header на публичных ручках не должно ломать запрос.
